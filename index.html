How it works

SEP for Web runs entirely in your browser. Files are read locally, processed with modern encryption, then saved back to your device, no uploads, no servers, no telemetry. You can download the app as a single sep.html file and use it offline.

Important notes

Security basics:
  • All encryption happens locally, you are responsible for managing passwords and keyfiles.
  • There is no password recovery, back up important data before encrypting.
  • There is no server rate limiting, a copied encrypted file can be tested many times offline.
  • SEP for Web is a university research project, no guarantees are provided about the implementation.
